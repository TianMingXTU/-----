### 第一阶段：迭代协议 (Iteration Protocol) —— 底层运转机制

这是所有 `for` 循环能够跑起来的基石。在 Python 中，不仅要会用，还要清楚它是怎么被调用的。

- **`__iter__(self)` (可迭代对象 Iterable)**
- **语法要点：** 只要一个类实现了这个魔法方法，它就是“可迭代的”。它的唯一职责是**返回一个迭代器对象**。

- **`__next__(self)` (迭代器 Iterator)**
- **语法要点：** 迭代器必须同时实现 `__iter__`（返回自身）和 `__next__`。每次调用 `__next__` 会返回数据流中的下一个元素。

- **`StopIteration` (终止信号)**
- **API解析：** 当数据耗尽时，`__next__` 必须抛出 `StopIteration` 异常。Python 的 `for` 循环在底层会自动捕获这个异常并优雅地结束循环。

- **`iter(obj)` 与 `next(iterator, default)` (内置触发器)**
- **API解析：** 这是我们手动触发迭代的内置函数。`iter(x)` 底层调用 `x.__iter__()`；`next(i)` 底层调用 `i.__next__()`，并且可以通过传入 `default` 参数来避免在数据耗尽时报错。

### 第二阶段：生成器基础 (Generators) —— 优雅的懒加载

生成器是迭代器的极致简化版，它让我们不用再去手写繁琐的 `__iter__` 和 `__next__` 类。

- **`yield` (状态挂起与产出)**
- **语法要点：** 只要函数内部出现了 `yield` 关键字，它就不再是普通函数，而是一个生成器函数（Generator Function）。调用它会返回一个生成器对象。执行到 `yield value` 时，函数会返回 `value` 并**冻结当前状态**（保留所有局部变量），直到下一次被 `next()` 唤醒。

- **`(expression for item in iterable)` (生成器表达式)**
- **语法要点：** 列表推导式 `[...]` 会一次性在内存中生成所有数据；而用圆括号 `(...)` 包裹则变成了生成器表达式。
- _实战场景：_ 读取几个G的大型日志文件或流式处理数据库结果时，用来节省内存。

### 第三阶段：高级协程交互 —— 双向通信

生成器不仅能往外“吐”数据，还能从外部“吃”进数据。这是早于 `async/await` 之前，Python 实现协程的古老但核心的机制。

- **`send(value)` (注入数据)**
- **API解析：** 恢复生成器的执行，并将 `value` 作为 `yield` 表达式的返回值赋给左边的变量（如 `res = yield 10`）。
- _注意：_ 刚启动的生成器必须先调用 `next(g)` 或 `g.send(None)` 才能推进到第一个 `yield` 处。

- **`throw(type, value, traceback)` (注入异常)**
- **API解析：** 在生成器内部 `yield` 挂起的地方抛出一个指定的异常。允许生成器内部使用 `try...except` 捕获并做清理工作。

- **`close()` (优雅关闭)**
- **API解析：** 在生成器挂起的地方引发 `GeneratorExit` 异常。常用于强制结束生成器的生命周期。

### 第四阶段：生成器委派 —— 管道化调度

当我们需要把多个生成器串联起来，或者把复杂的生成器拆解为子生成器时使用。

- **`yield from <iterable/generator>` (扁平化与委派)**
- **语法要点：** Python 3.3+ 引入。它等价于 `for item in iterable: yield item`，但远比这强大。
- **API解析：** 它能在调用者和子生成器之间建立一条**透明的双向管道**。外部调用的 `send()` 和 `throw()` 会直接传递给最底层的子生成器，无需我们在中间层写样板代码去转发。

### 第五阶段：实战工具箱 —— `itertools` 模块

Python 官方提供的高阶迭代器函数库，用 C 语言实现，性能极高。

- **`itertools.count(start, step)` (无限计数)**
- 生成无限的等差数列。

- **`itertools.cycle(iterable)` (无限循环)**
- 将传入的可迭代对象首尾相连，无限循环下去。

- **`itertools.chain(*iterables)` (无缝拼接)**
- 将多个不同的可迭代对象连在一起，像一个对象一样去遍历，而不需要在内存中拼接列表。

- **`itertools.islice(iterable, start, stop, step)` (切片)**
- 普通的切片 `[:]` 只能用于列表；对生成器切片必须用 `islice`，因为它通过消费数据来模拟切片，不会触发内存溢出。
