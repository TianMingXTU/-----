  写一段极简的 TCP 服务端 `while True` 接收数据的骨架。
- **红线验收**：展示你如何使用 `try...finally` 块（或者完美的结构流），保证无论中间 `recv` 是否抛出异常，`socket_ap.close()` 都绝对、一定会被执行！

```py
"""
from join where group by having select order by limit
"""

import functools
import time


def time_logger(t=None):
    def decorate(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            start_time = time.time()
            result = func(*args, **kwargs)
            print(f"耗时:{time.time()-start_time}")
            return result

        return wrapper

    return decorate


import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

server.bind(("127.0.0.1", 8888))

server.listen(5)

while True:
    try:
        socket_ap, socket_info = server.accept()
        result = socket_ap.recv(1024).decode("utf-8")
        if result == "exit":
            break
    finally:
        socket_ap.close()

server.close()
```

### 🧬 第四关：面向对象的灵魂协议（T-15节点：魔法方法重构）

**系统场景**：在主流的 AI 框架（如 PyTorch）中，我们经常把一个类的实例直接当成函数来调用，比如 `model = ResNet(); output = model(image)`。

- **硬核任务（魔法方法实操）：**
- 定义一个类 `DataTransformer`。
- 它的 `__init__` 接收一个参数 `scale`（缩放比例）并保存为实例属性。
- 重写**调用协议（特定的魔法方法）**，使得当你执行 `transformer_obj(5)` 时，它能直接返回 `5 * scale`。

```python
class DataTransformer:
    def __init__(self, scale):
        self.scale = scale

    def __call__(self, n):
        return n * self.scale


transformer_obj = DataTransformer(10)
print(transformer_obj(5))
```
