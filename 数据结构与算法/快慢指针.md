## 🧱 数组篇（核心思维：读写分离，`fast` 找目标，`slow` 负责写）

### 题目 1：移动零 (LeetCode 283)

**【场景描述】**
给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。

- _注意：必须在原数组上操作，不能拷贝额外数组。_
- 示例：输入 `[0, 1, 0, 3, 12]` $\rightarrow$ 输出 `[1, 3, 12, 0, 0]`

**【思考引导】**

1. **快指针 `fast` 的使命是什么？** 它负责去前面侦察，寻找所有“不是 0 的优秀元素”。
2. **慢指针 `slow` 的使命是什么？** 它代表当前“非零元素”应该写入的下一个位置。
3. **关键动作：** 当 `fast` 找到了一个非零元素，把它和 `slow` 指向的位置进行**交换（Swap）**，然后 `slow` 往前走一步。想一想，为什么用“交换”比用“覆盖”更好？（提示：用交换的话，0 会自然而然被滚到后面去）。

```py
def move_zore(num: List[int]):
    slow = 0
    for fast in range(len(num)):
        if num[fast] != 0:
            num[slow], num[fast] = num[fast], num[slow]
            slow += 1
    return num

```

## 🔗 链表篇（核心思维：速度差，`fast` 走两步，`slow` 走一步）

### 题目 3：环形链表 II - 寻找入环节点 (LeetCode 142)

**【场景描述】**
给定一个链表，返回链表**开始入环的第一个节点**。如果链表无环，则返回 `None`。

- 示例：链表 `1 -> 2 -> 3 -> 4 -> 2（回到2）`，你应该返回节点 `2` 的引用。

**【思考引导】**
这是一道非常经典的数学推导题，分为两步：

1. **第一步（判环）：** 用快慢指针（`fast` 走 2，`slow` 走 1），如果相遇，说明有环。
2. **第二步（找入口）：** 当它们在环内某点相遇时，**把其中一个指针（比如 `fast`）重新放回链表的头部（`head`）**，然后让 `fast` 和 `slow` **同时以相同的速度（每次只走 1 步）** 向前走。当它们再次相遇时，相遇的点就绝对是**入环的第一个节点**！
3. _你能试着用纸笔画一下，推导一下为什么“头节点到环入口的距离”等于“相遇点到环入口的距离”吗？_

### 题目 4：删除链表的倒数第 N 个节点 (LeetCode 19)

**【场景描述】**
给你一个链表，删除链表的倒数第 `n` 个节点，并且返回链表的头节点。

- 示例：链表 `1 -> 2 -> 3 -> 4 -> 5`，`n = 2`。倒数第 2 个是 `4`，删除后变为 `1 -> 2 -> 3 -> 5`。

**【思考引导】**
这道题的快慢指针不是“速度差”，而是“时间差（前后指针）”，但它们最终步调一致（都每次走 1 步）。

1. **如何定位倒数第 N 个？** 让 `fast` 指针先向前走 `n` 步。
2. 此时，`fast` 和 `slow`（在起点）之间就隔了 `n` 个节点的距离。
3. 接着，让 `fast` 和 `slow` 以相同的速度（每次 1 步）一起向前走。当 `fast` 走到链表末尾（`None`）时，`slow` 正好停留在哪里？（提示：正好停留在倒数第 N 个节点的前驱节点上，方便执行删除操作）。
