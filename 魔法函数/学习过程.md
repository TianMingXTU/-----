### 第一阶段：描述符协议 (Descriptor Protocol) —— 属性的终极接管

这是 Python 中最强大的特性之一（ORM框架、`@property` 的底层原理）。只要一个类实现了以下任意一个方法，它就是一个描述符。它用于**托管另一个类的类属性**。

- **`__set_name__(self, owner, name)` (Python 3.6+ 核心)**
- **触发时机：** 包含该描述符的类被创建时触发。
- **API 用法：** 自动获取描述符在宿主类中的变量名。非常关键，免去了过去在 `__init__` 中手动传入变量名的麻烦。

- **`__get__(self, instance, owner)` (拦截读取)**
- **触发时机：** 当通过宿主实例读取属性时（例如 `obj.attr`）。
- **API 用法：** `instance` 是宿主对象（如果是通过类名访问，则为 `None`），`owner` 是宿主类。通常返回计算后的值或托管的内部变量。

- **`__set__(self, instance, value)` (拦截赋值)**
- **触发时机：** 当给属性赋值时（例如 `obj.attr = 10`）。
- **API 用法：** 这里是做**数据类型校验**和**范围限制**的最佳地点。通常将拦截通过校验后的 `value` 存入 `instance.__dict__` 中。

- **`__delete__(self, instance)` (拦截删除)**
- **触发时机：** 使用 `del obj.attr` 时触发。

### 第二阶段：对象生命周期 —— 诞生与初始化

重点区分对象的“分配内存”与“数据初始化”。

- **`__new__(cls, *args, kwargs)` (真正的构造函数)**
- **触发时机：** 实例创建的第一步，早于 `__init__`。
- **API 用法：** 必须返回一个实例对象（通常是 `super().__new__(cls)`）。
- _实战场景：_ 实现单例模式（Singleton），或者继承不可变类型（如 `tuple` 或 `str`）时修改其初始值。

- **`__init__(self, *args, kwargs)` (初始化函数)**
- **触发时机：** `__new__` 返回实例后立刻触发。
- **API 用法：** 纯粹用于给 `self` 绑定初始属性，**只能返回 `None**`。

### 第三阶段：动态属性拦截 —— 兜底与全局控制

当描述符不够用，需要对实例的*所有*属性进行泛化拦截时使用。**这里是初学者最容易写出无限递归死循环（RecursionError）的雷区。**

- **`__getattr__(self, name)` (优雅的兜底)**
- **触发时机：** **只有**当常规途径找不到属性，且描述符也没有这个属性时，才会作为最后手段触发。
- _实战场景：_ 实现链式调用（如处理动态生成的 API 路径）。

- **`__getattribute__(self, name)` (绝对的独裁者)**
- **触发时机：** 只要访问任何属性，**无条件第一优先触发**。
- **API 用法：** 极易引发无限递归。如果在里面使用 `self.name`，会再次触发自身。必须通过 `super().__getattribute__(name)` 来获取值。

- **`__setattr__(self, name, value)` (全局赋值拦截)**
- **触发时机：** 实例级别任何 `self.xxx = yyy` 都会触发。
- **API 用法：** 同样有递归陷阱。赋值时必须写成 `super().__setattr__(name, value)` 或操作 `self.__dict__[name] = value`。

### 第四阶段：行为对象化 —— 让对象动起来

让普通的类实例拥有类似函数、字典或上下文管理器的行为。

- **`__call__(self, *args, kwargs)` (可调用对象)**
- **触发时机：** 像调用函数一样调用实例时（例如 `obj()`）。
- _实战场景：_ 编写带状态的装饰器，或者替代闭包，让代码结构更清晰。

- **`__getitem__(self, key)` / `__setitem__(self, key, value)` (字典/列表模拟)**
- **触发时机：** 使用中括号语法时（例如 `obj['key']` 或 `obj[0] = 1`）。
- _实战场景：_ 构建自定义的数据容器、配置管理类。

- **`__enter__(self)` / `__exit__(self, exc_type, exc_val, exc_tb)` (上下文管理)**
- **触发时机：** 与 `with` 语法绑定。
- **API 用法：** `__enter__` 返回要使用的对象，`__exit__` 负责清理资源（如关闭文件、回滚数据库），并可以通过返回 `True` 来吞掉内部抛出的异常。

### 第五阶段：字符串表现形式 —— 调试与输出

控制对象在打印时的表现形态。

- **`__str__(self)` (面向用户)**
- **触发时机：** 被 `print()` 或 `str()` 调用时。
- **API 用法：** 返回人类易读的字符串。

- **`__repr__(self)` (面向开发者)**
- **触发时机：** 在交互式解释器中直接敲击变量名，或被 `repr()` 调用时。若没有定义 `__str__`，`print()` 会退化调用它。
- **API 用法：** 返回的值最好能直接复制粘贴用来重新创建这个对象（例如返回 `MyClass(x=1, y=2)`）。
